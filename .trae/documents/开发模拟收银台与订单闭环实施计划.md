# 第13步：开发模拟收银台与订单闭环

## 1. 后端增强 (Backend Core)
### 目标
补充订单详情查询接口，用于收银台数据回显。

### 任务
1.  **修改 `OrderService` 接口**：
    *   新增 `Order getByOrderNo(String orderNo)` 方法。
2.  **修改 `OrderServiceImpl`**：
    *   实现 `getByOrderNo`，调用 `OrderMapper.selectByOrderNo`。
    *   增加校验：确保查询到的订单属于当前登录用户（防止越权）。
3.  **修改 `OrderController`**：
    *   新增 `GET /getByNo` 接口，接收 `orderNo` 参数。
    *   调用 Service 并返回 `Result<Order>`。

## 2. 前端页面开发 (Frontend Views)
### 目标
实现“订单确认 -> 收银台 -> 订单列表”的完整C端业务闭环。

### 任务
1.  **开发 `OrderConfirm.vue` (订单确认页)**：
    *   **布局**：左侧商品清单，右侧金额结算卡片。
    *   **数据**：初始化调用 `/cart/list` 获取待结算商品。
    *   **逻辑**：点击“提交订单”调用 `/order/create`，成功后获取 `orderNo`，跳转 `/cashier?orderNo=xxx`。
    *   **样式**：使用“多巴胺”配色（背景奶茶色，卡片白色圆角）。
2.  **开发 `Cashier.vue` (收银台)**：
    *   **数据**：初始化调用 `/order/getByNo` 获取订单金额。
    *   **UI**：展示超大金额数字（橙色），倒计时（15:00），支付方式选择（微信/支付宝/余额）。
    *   **交互**：点击“立即支付” -> 全屏 Loading (模拟支付) -> 1.5s 后提示成功 -> 跳转 `/order`。
    *   **样式**：支付方式选中态使用“果汁软糖”渐变色。

## 3. 路由与联动 (Router & Logic)
### 目标
配置路由并串联业务流程。

### 任务
1.  **修改 `router/index.js`**：
    *   注册 `/confirm` (OrderConfirm) 和 `/cashier` (Cashier)。
    *   配置 `meta: { requiresAuth: true }`。
2.  **修改 `Cart.vue`**：
    *   将“立即结算”按钮的点击事件，从直接调 API 改为 `router.push('/confirm')`。
