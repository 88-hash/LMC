# 第16步：恢复核销中心默认显示逻辑

## 1. 核心目标
响应用户反馈：“核销中心看不到了未核销的订单...你给我正常的核销中心搞错误了”。
用户希望 **核销中心 (Verify.vue)** 恢复为 **默认显示所有待核销订单**（方便店长直接操作），而 **订单管理 (AdminOrders.vue)** 保持 **默认不显示**（避免数据量大，需查询）。

## 2. 前端修改 (Verify.vue)
### 任务
1.  **移除“空状态”逻辑**：
    *   删除 `v-if="!hasSearched"` 判断，让表格始终显示。
    *   删除 `empty-state` 相关的 div。
2.  **恢复初始化查询**：
    *   在 `onMounted` 中直接调用 `fetchData`（或 `handleSearch` 的核心逻辑）。
    *   **重要**：移除 `handleSearch` 中对 `keyword` 的非空校验，允许空参数查询（即查询所有）。
3.  **优化统计逻辑**：
    *   将顶部卡片的数字绑定为 `allPendingList.length`（全量待核销），而非过滤后的 `filteredList.length`。
    *   这样即使用户在搜索某个特定订单，顶部的“待核销总数”依然准确显示当前积压总量。

## 3. 逻辑验证
*   **Order Center**: 保持现状 -> 进页面为空 -> 输入条件 -> 查询出结果。
*   **Verify Center**: 恢复旧逻辑 -> 进页面自动加载所有待核销 -> 顶部显示总数 -> 列表显示所有 -> 输入关键词可过滤。
